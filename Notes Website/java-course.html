1. Introduction to Java

What is Java?

Java is a high-level, object-oriented programming language developed by James Gosling at Sun Microsystems (now owned by Oracle) in 1995. Java was designed with the philosophy of "Write Once, Run Anywhere" (WORA), meaning that compiled Java code can run on all platforms that support Java without the need for recompilation.

Java is a compiled and interpreted language. Source code is compiled into bytecode, which is then executed by the Java Virtual Machine (JVM). This architecture makes Java platform-independent, secure, and portable. Java is one of the most popular programming languages in the world, widely used for developing web applications, mobile apps (Android), enterprise software, and large-scale systems.

Key Characteristics of Java:

  • Platform Independent: WORA - Write Once, Run Anywhere through JVM
  • Object-Oriented: Pure OOP language with classes and objects
  • Simple: Easy to learn with clear syntax
  • Secure: Built-in security features and sandbox execution
  • Multithreaded: Built-in support for concurrent programming
  • Robust: Strong memory management and exception handling
  • Portable: Architecture-neutral bytecode
  • High Performance: Just-In-Time (JIT) compilation
  • Distributed: Network-centric programming support
  • Dynamic: Runtime polymorphism and dynamic loading

Why Learn Java?

  • Industry Demand: One of the most sought-after programming skills
  • Android Development: Primary language for Android app development
  • Enterprise Applications: Widely used in large-scale enterprise systems
  • Career Opportunities: High-paying jobs in software development
  • Large Community: Extensive support and resources available
  • Rich Ecosystem: Vast libraries and frameworks (Spring, Hibernate, etc.)
  • Scalability: Excellent for building scalable applications

Your First Java Program

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Explanation:

  • public class HelloWorld - Defines a public class named HelloWorld (filename must match class name)
  • public static void main(String[] args) - Main method where program execution begins
  • public - Access modifier (accessible from anywhere)
  • static - Can be called without creating an object
  • void - Returns nothing
  • String[] args - Command-line arguments
  • System.out.println() - Prints text to console and adds a new line

Java Architecture:

  • JDK (Java Development Kit): Development environment for Java
  • JRE (Java Runtime Environment): Runtime environment to run Java programs
  • JVM (Java Virtual Machine): Virtual machine that executes bytecode
  • JIT (Just-In-Time) Compiler: Compiles bytecode to native code at runtime

2. Variables & Data Types

What are Variables?

Variables are containers that store data values in Java. In Java, variables must be declared with a specific data type before they can be used. Java is a strongly-typed language, meaning variable types are checked at compile time.

Variable Declaration Syntax:

data_type variable_name;
data_type variable_name = value;

Type Casting in Java

Java supports two types of casting:

  • Widening Casting (Implicit): Converting smaller type to larger type
  • Narrowing Casting (Explicit): Converting larger type to smaller type
// Widening casting
int num = 10;
double d = num;  // Automatic

// Narrowing casting
double d = 10.5;
int num = (int) d;  // Manual casting required

4. Control Flow

If-Else Statement

The if-else statement allows you to execute different code blocks based on conditions.

Example: If-Else

public class IfElseExample {
    public static void main(String[] args) {
        int number = 10;
        
        if (number > 0) {
            System.out.println("Number is positive");
        } else if (number < 0) {
            System.out.println("Number is negative");
        } else {
            System.out.println("Number is zero");
        }
    }
}

Switch Statement

The switch statement allows you to select one of many code blocks to execute based on a value.

Example: Switch

public class SwitchExample {
    public static void main(String[] args) {
        int day = 3;
        
        switch (day) {
            case 1:
                System.out.println("Monday");
                break;
            case 2:
                System.out.println("Tuesday");
                break;
            case 3:
                System.out.println("Wednesday");
                break;
            default:
                System.out.println("Invalid day");
        }
    }
}

Loops in Java

1. For Loop

public class ForLoopExample {
    public static void main(String[] args) {
        for (int i = 1; i <= 5; i++) {
            System.out.print(i + " ");
        }
        // Output: 1 2 3 4 5
    }
}

2. While Loop

public class WhileLoopExample {
    public static void main(String[] args) {
        int i = 1;
        while (i <= 5) {
            System.out.print(i + " ");
            i++;
        }
    }
}

3. Do-While Loop

public class DoWhileLoopExample {
    public static void main(String[] args) {
        int i = 1;
        do {
            System.out.print(i + " ");
            i++;
        } while (i <= 5);
    }
}

4. Enhanced For Loop (For-Each)

public class ForEachExample {
    public static void main(String[] args) {
        int[] numbers = {1, 2, 3, 4, 5};
        
        for (int num : numbers) {
            System.out.print(num + " ");
        }
    }
}

Jump Statements

  • break: Exits the loop or switch statement
  • continue: Skips the rest of the loop and continues to the next iteration
  • return: Exits the method

5. Methods

What are Methods?

Methods are blocks of code that perform a specific task. In Java, methods are defined within classes and are used to organize code into reusable blocks. Methods help in code reusability, modularity, and organization.

Method Syntax

access_modifier return_type method_name(parameters) {
    // method body
    return value;
}

Example: Simple Method

public class MethodsExample {
    // Method definition
    public static int add(int a, int b) {
        return a + b;
    }
    
    public static void main(String[] args) {
        int result = add(5, 3);
        System.out.println("Sum: " + result);
    }
}

Method Overloading

Java allows multiple methods with the same name but different parameters (method overloading).

public class MethodOverloading {
    // Overloaded methods
    public static int add(int a, int b) {
        return a + b;
    }
    
    public static double add(double a, double b) {
        return a + b;
    }
    
    public static int add(int a, int b, int c) {
        return a + b + c;
    }
    
    public static void main(String[] args) {
        System.out.println(add(5, 3));        // Calls int add(int, int)
        System.out.println(add(5.5, 3.2));    // Calls double add(double, double)
        System.out.println(add(1, 2, 3));     // Calls int add(int, int, int)
    }
}

Access Modifiers

  • public: Accessible from anywhere
  • private: Accessible only within the class
  • protected: Accessible within the package and subclasses
  • default (no modifier): Accessible within the package

Static vs Non-Static Methods

  • Static methods: Belong to the class, called without creating an object
  • Non-static methods: Belong to the instance, called using object reference

6. Object-Oriented Programming

What is OOP?

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of "objects" that contain data (attributes) and code (methods). Java is a pure OOP language, meaning everything in Java is an object.

Key OOP Concepts:

  • Class: Blueprint for creating objects
  • Object: Instance of a class
  • Encapsulation: Bundling data and methods together, hiding implementation details
  • Inheritance: Creating new classes from existing ones (extends keyword)
  • Polymorphism: Same interface, different implementations (method overriding/overloading)
  • Abstraction: Hiding implementation details (abstract classes and interfaces)

Class and Object Example

public class Student {
    // Attributes (fields)
    private String name;
    private int age;
    
    // Constructor
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    // Getter methods
    public String getName() {
        return name;
    }
    
    public int getAge() {
        return age;
    }
    
    // Setter methods
    public void setName(String name) {
        this.name = name;
    }
    
    public void setAge(int age) {
        this.age = age;
    }
    
    // Display method
    public void display() {
        System.out.println("Name: " + name + ", Age: " + age);
    }
    
    public static void main(String[] args) {
        // Creating objects
        Student s1 = new Student("John", 20);
        Student s2 = new Student("Alice", 22);
        
        s1.display();
        s2.display();
    }
}

Inheritance Example

// Base class
class Animal {
    public void eat() {
        System.out.println("Animal is eating");
    }
}

// Derived class
class Dog extends Animal {
    public void bark() {
        System.out.println("Dog is barking");
    }
}

public class InheritanceExample {
    public static void main(String[] args) {
        Dog myDog = new Dog();
        myDog.eat();   // Inherited from Animal
        myDog.bark();  // Own method
    }
}

Access Modifiers

  • public: Accessible from anywhere
  • private: Accessible only within the class
  • protected: Accessible within the package and subclasses
  • default (no modifier): Accessible within the package

7. Collections Framework

What is Collections Framework?

The Java Collections Framework is a unified architecture for representing and manipulating collections. It provides interfaces, implementations, and algorithms to work with data structures like lists, sets, maps, etc.

Main Interfaces:

  • List: Ordered collection that allows duplicates (ArrayList, LinkedList)
  • Set: Collection that does not allow duplicates (HashSet, TreeSet)
  • Map: Key-value pairs (HashMap, TreeMap)
  • Queue: FIFO data structure (PriorityQueue, LinkedList)

1. ArrayList (Dynamic Array)

import java.util.ArrayList;

public class ArrayListExample {
    public static void main(String[] args) {
        ArrayList<String> list = new ArrayList<>();
        
        // Adding elements
        list.add("Apple");
        list.add("Banana");
        list.add("Orange");
        
        // Accessing elements
        System.out.println(list.get(0));
        System.out.println("Size: " + list.size());
        
        // Iterating
        for (String fruit : list) {
            System.out.println(fruit);
        }
    }
}

2. HashMap (Key-Value Pairs)

import java.util.HashMap;

public class HashMapExample {
    public static void main(String[] args) {
        HashMap<String, Integer> map = new HashMap<>();
        
        map.put("Apple", 10);
        map.put("Banana", 20);
        map.put("Orange", 15);
        
        System.out.println(map.get("Apple"));
        
        // Iterating
        for (String key : map.keySet()) {
            System.out.println(key + ": " + map.get(key));
        }
    }
}

3. HashSet (Unique Elements)

import java.util.HashSet;

public class HashSetExample {
    public static void main(String[] args) {
        HashSet<String> set = new HashSet<>();
        
        set.add("Apple");
        set.add("Banana");
        set.add("Apple");  // Duplicate, will be ignored
        
        System.out.println("Size: " + set.size());
        
        for (String item : set) {
            System.out.println(item);
        }
    }
}

8. Exception Handling

What is Exception Handling?

Exception handling is a mechanism to handle runtime errors in Java. An exception is an event that disrupts the normal flow of program execution. Java provides try-catch blocks to handle exceptions gracefully.

Types of Exceptions:

  • Checked Exceptions: Compile-time exceptions that must be handled (IOException, SQLException)
  • Unchecked Exceptions: Runtime exceptions (NullPointerException, ArrayIndexOutOfBoundsException)
  • Errors: Serious problems that are not recoverable (OutOfMemoryError, StackOverflowError)

Try-Catch Block

public class ExceptionExample {
    public static void main(String[] args) {
        try {
            int result = 10 / 0;  // This will throw ArithmeticException
            System.out.println("Result: " + result);
        } catch (ArithmeticException e) {
            System.out.println("Error: Division by zero!");
            System.out.println("Exception: " + e.getMessage());
        } finally {
            System.out.println("This block always executes");
        }
    }
}

Multiple Catch Blocks

public class MultipleCatchExample {
    public static void main(String[] args) {
        try {
            int[] numbers = {1, 2, 3};
            System.out.println(numbers[5]);  // ArrayIndexOutOfBoundsException
            int result = 10 / 0;  // ArithmeticException
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("Array index out of bounds!");
        } catch (ArithmeticException e) {
            System.out.println("Division by zero!");
        } catch (Exception e) {
            System.out.println("General exception: " + e.getMessage());
        }
    }
}

Throw Keyword

public class ThrowExample {
    public static void validateAge(int age) {
        if (age < 18) {
            throw new ArithmeticException("Age must be 18 or above");
        } else {
            System.out.println("Age is valid");
        }
    }
    
    public static void main(String[] args) {
        validateAge(15);  // This will throw an exception
    }
}

Throws Keyword

import java.io.IOException;

public class ThrowsExample {
    public static void readFile() throws IOException {
        // Code that may throw IOException
        throw new IOException("File not found");
    }
    
    public static void main(String[] args) {
        try {
            readFile();
        } catch (IOException e) {
            System.out.println("Exception handled: " + e.getMessage());
        }
    }
}
