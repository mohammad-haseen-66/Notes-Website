1. Introduction to HTML5

What is HTML5?

HTML5 is the fifth and current version of the HyperText Markup Language, the standard markup language for creating web pages and web applications.

Key Features:

  • Semantic Elements: Better structure with semantic tags
  • Media Support: Built-in audio and video support
  • Canvas: 2D graphics and animations
  • Local Storage: Store data in the browser
  • Offline Support: Work offline with web apps

Basic HTML5 Structure

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My First Page</title>
</head>
<body>
    <h1>Hello, World!</h1>
</body>
</html>

2. HTML Elements

What are HTML Elements?

HTML elements are the building blocks of web pages. An HTML element consists of a start tag, content, and an end tag. Some elements are self-closing and don't require an end tag.

Basic Structure of an HTML Element

<tagname>Content goes here</tagname>

/* Example */
<p>This is a paragraph</p>

Heading Elements

Headings define the structure and hierarchy of your content. HTML provides six levels of headings:

<h1>Heading 1 - Most Important</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6 - Least Important</h6>

Block and Inline Elements

Block elements take up the full width available and start on a new line:

<p>Paragraph is block</p>
<h1>Heading is block</h1>
<ul>List is block</ul>

Inline elements only take up as much width as needed and don't start on a new line:

<span>Inline element</span>
<a href="#">Link is inline</a>
<strong>Strong is inline</strong>
<img src="image.jpg" alt="Image is inline">

Div and Span Elements

</div>

<!-- Span: inline container -->
<p>This is a <span class="highlight">highlighted</span> word.</p>

3. HTML Attributes

What are HTML Attributes?

HTML attributes provide additional information about HTML elements. They are always specified in the start tag and usually come in name/value pairs like: name="value".

Common Global Attributes

These attributes can be used on any HTML element:

id Attribute

Specifies a unique identifier for an element:

<div id="header">Header content</div>
<div id="main-content">Main content</div>
<div id="footer">Footer content</div>

class Attribute

Specifies one or more class names for an element (used for CSS styling):

<p class="intro">Introduction paragraph</p>
<p class="highlight important">Multiple classes</p>
<div class="container">Container div</div>

style Attribute

Allows inline CSS styling:

<p style="color: red; font-size: 20px;">Styled paragraph</p>
<div style="background-color: blue; padding: 10px;">Styled div</div>

title Attribute

Provides additional information about an element (shown as tooltip):

<p title="This is a tooltip">Hover over me</p>
<abbr title="HyperText Markup Language">HTML</abbr>

lang Attribute

Specifies the language of the element's content:

<html lang="en">
<p lang="fr">Bonjour le monde</p>
<p lang="es">Hola mundo</p>

data-* Attributes

Custom data attributes for storing extra information:

<div data-user-id="123" data-role="admin">User content</div>
<button data-action="delete" data-item-id="456">Delete</button>

Element-Specific Attributes

Anchor Tag Attributes

<!-- href: specifies the URL -->
<a href="https://example.com">Link</a>

<!-- target: where to open the link -->
<a href="page.html" target="_blank">Open in new tab</a>
<a href="page.html" target="_self">Open in same tab</a>

<!-- rel: relationship between current and linked document -->
<a href="external.html" rel="noopener noreferrer">External link</a>

<!-- download: downloads the file -->
<a href="file.pdf" download>Download PDF</a>

Image Tag Attributes

<!-- src: image source URL -->
<img src="image.jpg" alt="Description">

<!-- alt: alternative text (required for accessibility) -->
<img src="photo.jpg" alt="A beautiful sunset">

<!-- width and height: image dimensions -->
<img src="image.jpg" alt="Description" width="300" height="200">

<!-- loading: lazy loading -->
<img src="image.jpg" alt="Description" loading="lazy">

<!-- srcset: responsive images -->
<img src="small.jpg" 
     srcset="small.jpg 300w, medium.jpg 600w, large.jpg 1200w"
     sizes="(max-width: 600px) 300px, (max-width: 1200px) 600px, 1200px"
     alt="Responsive image">

Input Tag Attributes

<!-- type: input type -->
<input type="text" name="username">
<input type="email" name="email">
<input type="password" name="password">
<input type="number" name="age">
<input type="date" name="birthday">
<input type="checkbox" name="agree">
<input type="radio" name="gender">

<!-- placeholder: hint text -->
<input type="text" placeholder="Enter your name">

<!-- required: makes field mandatory -->
<input type="email" required>

<!-- disabled: disables the input -->
<input type="text" disabled>

<!-- readonly: makes input read-only -->
<input type="text" readonly value="Cannot edit">

<!-- maxlength: maximum characters -->
<input type="text" maxlength="50">

<!-- min and max: for number/date inputs -->
<input type="number" min="0" max="100">
<input type="date" min="2020-01-01" max="2025-12-31">

Table Tag Attributes

<table border="1">
    <tr>
        <td colspan="2">Spans 2 columns</td>
    </tr>
    <tr>
        <td rowspan="2">Spans 2 rows</td>
        <td>Cell</td>
    </tr>
</table>

Meta Tag Attributes

<!-- Character encoding -->
<meta charset="UTF-8">

<!-- Viewport for responsive design -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

<!-- Description for SEO -->
<meta name="description" content="Page description">

<!-- Keywords (less important now) -->
<meta name="keywords" content="HTML, CSS, JavaScript">

<!-- Author -->

<!-- Open Graph for social media -->
<meta property="og:title" content="Page Title">
<meta property="og:description" content="Page Description">

Boolean Attributes

Some attributes don't need a value - their presence alone indicates true:

<input type="text" disabled>
<input type="text" readonly>
<option selected>Selected option</option>
<video controls>Video content</video>

4. Forms

What are HTML Forms?

HTML forms are used to collect user input. Forms contain form elements like text fields, checkboxes, radio buttons, submit buttons, etc. The data collected is usually sent to a server for processing.

Form Element

The <form> element is a container for form elements:

<form action="/submit" method="POST">
    <!-- Form elements go here -->
</form>

Form Attributes

  • action: Specifies where to send form data
  • method: Specifies HTTP method (GET or POST)
  • enctype: Specifies how form data should be encoded
  • target: Specifies where to display response
  • novalidate: Disables validation

Input Elements

The <input> element is the most important form element. It can be displayed in many ways depending on the type attribute:

<input type="text" name="username" placeholder="Enter username">
<input type="text" name="fullname" value="John Doe">

5. Semantic HTML

What is Semantic HTML?

Semantic HTML uses HTML elements that have meaning and clearly describe their purpose. Instead of using generic <div> and <span> elements, semantic HTML uses elements like <header>, <nav>, <article>, etc., which clearly indicate their role in the document.

Benefits of Semantic HTML

  • Accessibility: Screen readers can better understand page structure
  • SEO: Search engines can better index content
  • Maintainability: Code is easier to read and maintain
  • Future-proof: Better compatibility with future technologies

Semantic Elements

<header>

Represents introductory content or navigational aids:

<header>
    <h1>Website Title</h1>
    <nav>
        <a href="#">Home</a>
        <a href="#">About</a>
    </nav>
</header>

<nav>

Defines navigation links:

<nav>
    <ul>
        <li><a href="#">Home</a></li>
        <li><a href="#">About</a></li>
        <li><a href="#">Contact</a></li>
    </ul>
</nav>

<main>

Represents the main content of the document (should be unique):

<main>
    <h1>Main Content</h1>
    <p>This is the main content area.</p>
</main>

<article>

Represents independent, self-contained content:

<article>
    <header>
        <h2>Article Title</h2>
        <p>Published on <time datetime="2025-01-15">January 15, 2025</time></p>
    </header>
    <p>Article content goes here...</p>
    <footer>
        <p>Author: John Doe</p>
    </footer>
</article>

<section>

Represents a thematic grouping of content:

<section>
    <h2>Introduction</h2>
    <p>This section introduces the topic.</p>
</section>

<section>
    <h2>Main Content</h2>
    <p>This section contains the main content.</p>
</section>

<aside>

Represents content aside from the main content (sidebars, call-out boxes):

<aside>
    <h3>Related Articles</h3>
    <ul>
        <li><a href="#">Article 1</a></li>
        <li><a href="#">Article 2</a></li>
    </ul>
</aside>

Represents footer content:

    <p>Â© 2025 Company Name. All rights reserved.</p>

<figure> and <figcaption>

Used for images, diagrams, code blocks with captions:

<figure>
    <img src="image.jpg" alt="Description">
    <figcaption>This is a caption for the image</figcaption>
</figure>

<figure>
    <pre><code>function example() {
    return "Hello";
}</code></pre>
    <figcaption>Example JavaScript code</figcaption>
</figure>

<time>

Represents a specific period in time:

<time datetime="2025-01-15">January 15, 2025</time>
<time datetime="2025-01-15T10:30">10:30 AM on January 15</time>
<time datetime="PT2H30M">2 hours 30 minutes</time>

<mark>

Highlights text for reference:

<p>This is <mark>important</mark> information.</p>

<address>

Represents contact information:

<address>
    Written by <a href="mailto:author@example.com">John Doe</a><br>

    123 Main St<br>

6. Media Elements

What are Media Elements?

HTML5 provides native support for audio and video content without requiring plugins. These elements allow you to embed multimedia content directly into web pages.

Audio Element

The <audio> element is used to embed sound content:

<!-- Basic audio -->
<audio src="audio.mp3" controls></audio>

<!-- Audio with multiple sources -->
<audio controls>
    <source src="audio.mp3" type="audio/mpeg">
    <source src="audio.ogg" type="audio/ogg">
    Your browser does not support the audio element.
</audio>

<!-- Audio with all attributes -->
<audio controls autoplay loop muted preload="auto">
    <source src="audio.mp3" type="audio/mpeg">
    Your browser does not support the audio element.
</audio>

Audio Attributes

  • controls: Displays audio controls (play, pause, volume)
  • autoplay: Starts playing automatically (use with caution)
  • loop: Repeats audio when finished
  • muted: Mutes audio by default
  • preload: Specifies if audio should be preloaded (auto, metadata, none)
  • src: URL of audio file

Video Element

The <video> element is used to embed video content:

<!-- Basic video -->
<video src="video.mp4" controls></video>

<!-- Video with multiple sources -->
<video controls width="640" height="360">
    <source src="video.mp4" type="video/mp4">
    <source src="video.webm" type="video/webm">
    <source src="video.ogg" type="video/ogg">
    Your browser does not support the video element.
</video>

<!-- Video with poster image -->
<video controls poster="poster.jpg">
    <source src="video.mp4" type="video/mp4">
</video>

<!-- Video with all attributes -->
<video 
    controls 
    autoplay 
    loop 
    muted 
    preload="auto"
    width="800"
    height="450"
    poster="poster.jpg">
    <source src="video.mp4" type="video/mp4">
    Your browser does not support the video element.
</video>

Video Attributes

  • controls: Displays video controls
  • autoplay: Starts playing automatically
  • loop: Repeats video when finished
  • muted: Mutes video by default
  • preload: Specifies if video should be preloaded
  • poster: Image shown before video plays
  • width: Video width
  • height: Video height
  • src: URL of video file

Iframe Element

The <iframe> element embeds another HTML page within the current page:

<!-- Basic iframe -->
<iframe src="https://example.com"></iframe>

<!-- Iframe with dimensions -->
<iframe src="page.html" width="800" height="600"></iframe>

<!-- Iframe with attributes -->
<iframe 
    src="https://example.com"
    width="800"
    height="600"
    frameborder="0"
    allowfullscreen
    sandbox="allow-scripts allow-same-origin">
</iframe>

<!-- Embedding YouTube video -->
<iframe 
    width="560" 
    height="315" 
    src="https://www.youtube.com/embed/VIDEO_ID" 
    frameborder="0" 
    allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture" 
    allowfullscreen>
</iframe>

Embed Element

The <embed> element embeds external content:

<embed src="file.pdf" type="application/pdf" width="800" height="600">
<embed src="animation.swf" type="application/x-shockwave-flash">

Object Element

The <object> element embeds external resources:

<object data="file.pdf" type="application/pdf" width="800" height="600">
    <p>Alternative text if object cannot be displayed</p>
</object>

Picture Element

The <picture> element provides responsive images:

<picture>
    <source media="(min-width: 1200px)" srcset="large.jpg">
    <source media="(min-width: 768px)" srcset="medium.jpg">
    <img src="small.jpg" alt="Responsive image">
</picture>

<picture>
    <source srcset="image.webp" type="image/webp">
    <source srcset="image.jpg" type="image/jpeg">
    <img src="image.jpg" alt="Fallback image">
</picture>

Canvas Element

The <canvas> element is used to draw graphics via JavaScript:

<canvas id="myCanvas" width="400" height="200">
    Your browser does not support the canvas element.
</canvas>

<script>
    const canvas = document.getElementById('myCanvas');
    const ctx = canvas.getContext('2d');
    ctx.fillStyle = 'red';
    ctx.fillRect(10, 10, 100, 100);
</script>

SVG (Scalable Vector Graphics)

SVG can be embedded directly in HTML:

<!-- Inline SVG -->
<svg width="100" height="100">
    <circle cx="50" cy="50" r="40" stroke="black" stroke-width="2" fill="red" />
</svg>

<!-- SVG as image -->
<img src="image.svg" alt="SVG image">

<!-- SVG as object -->
<object data="image.svg" type="image/svg+xml"></object>

<!DOCTYPE html>
<html>

    <h1>Audio Example</h1>

        Your browser does not support audio.
    </audio>
    
    <h1>Video Example</h1>
    <video controls width="640" height="360" poster="poster.jpg">
        <source src="video.mp4" type="video/mp4">
        Your browser does not support video.
    </video>
    
    <h1>Canvas Example</h1>
    <canvas id="myCanvas" width="400" height="200"></canvas>
    <script>
        const canvas = document.getElementById('myCanvas');
        const ctx = canvas.getContext('2d');
        ctx.fillStyle = 'blue';
        ctx.fillRect(50, 50, 100, 100);
    </script>
</body>
</html>
